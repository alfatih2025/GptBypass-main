1项目简介
一个基于 FastAPI 的 OpenAI 兼容 API 本地代理工具，GUI 采用绿色主题界面，单文件绿色免安装运行。
它解决什么问题？

在使用 GPT 等大模型进行逆向分析、安全研究等场景时，模型经常会以"道德限制"为由拒绝回答。本工具作为本地代理中间层，自动检测模型拒答并触发智能改写重试，帮助你获取所需的技术协助。

本地代理

运行在 127.0.0.1:8999，客户端只需将 API 地址指向本地即可，无需修改客户端代码。

拒答检测

通过可配置的关键词列表，自动识别模型回复中的拒答信号（如"抱歉""I'm sorry"）。

智能改写

调用辅助模型对用户消息进行自然重写，保留核心意图，避开触发拒答的表述方式。

自动重试

改写后自动重新发送请求，最多重试 5 次（可配置），直到获得有效回复。

绿色免安装

PyInstaller 打包为单文件 EXE，无需安装 Python 环境，双击即用，配置文件同目录存放。

Rust 加速

内置 Rust Aho-Corasick 关键词匹配模块，关键词过滤性能远超纯 Python 实现。

3核心功能
工具提供的主要能力一览。
请求转发与伪造
  • 全量转发 — 所有 /v1/* 请求原样转发到目标模型接口
  • 本地伪造 /v1/models — 返回静态模型列表，方便客户端做连通性检查
  • 流式支持 — 完整支持 SSE 流式响应，实时返回给客户端
请求分类与过滤
  • 主/辅请求识别 — 自动区分主请求和标题生成、摘要等辅助请求
  • 辅助请求可跳过 — 辅助请求直接透传，不进入改写链路
  • HTTP 头控制 — 支持 x-jmp-main-request、x-jmp-skip-optimize 等头部手动控制
接口测试

GUI 内置目标接口和改写接口的实时测试按钮，一键发送测试请求（"请回复：模型测试成功"），立即验证接口地址和密钥是否正确，无需借助第三方工具。

关于

目标模型选项卡

配置最终接收请求的目标模型接口信息。

  • 目标模型名称转发到目标后端时使用的模型名称，如 gpt-5.4
  • 消息接口类型API 类型标识：responses 或 chat.completions
  • 推理深度目标模型推理工作量级别，如 high / medium / low
  • 目标接口地址目标 API 端点，建议填写完整地址到 /v1，右侧有【测试】按钮
  • 目标接口密钥API Key，可点击眼睛图标切换显示/隐藏，留空则不发送 Authorization 头
右侧面板展示当前部署概览：监听地址、目标模型、消息策略、配置文件路径，以及快捷保存和打开目录按钮。

改写模型选项卡

配置拒答后用于重写用户消息的辅助模型。

  • 改写模型名称辅助模型名称，如 minimaxai/minimax-m2.7
  • 改写接口地址辅助模型 API 地址，右侧有【测试】按钮
  • 改写接口密钥辅助模型 API Key
  • 记录完整改写开关，启用后在日志中记录辅助模型完整的改写结果
  • 仅改写主请求开关，启用（默认）时仅对主用户请求启用改写（标题生成、摘要等辅助请求直接透传）；关闭时所有请求均进入改写链路
  • System Prompt右侧大文本框，可直接编辑改写模型的系统提示词，支持复制和清空

响应过滤选项卡

管理拒答检测的关键词列表。

  • 拦截词列表左侧文本框，每行一个关键词。仅检查最后一条模型回复，命中后丢弃并自动重试
默认内置中英文拒答词：抱歉 不能协助 无法提供 I'm sorry I cannot assist as an AI 等，可根据目标模型特点自由增减。

运行日志选项卡

实时查看代理运行状态，排查问题。

  • 实时日志持续读取本地 proxy.log，显示请求转发、过滤命中、改写重试等运行信息
  • 复制日志一键复制当前日志内容到剪贴板
  • 清空显示清空当前日志显示（不删除日志文件）

6快速上手
三步开始使用。
Step 1 - 配置目标模型

打开 GUI，在目标模型选项卡中填入：

  • 目标模型名称（如 gpt-5.4）
  • 目标接口地址（写到 /v1）
  • 目标接口密钥

点击接口地址右侧的【测试】按钮验证连通性。

Step 2 - 配置改写模型

切换到改写模型选项卡，填入辅助模型信息：

  • 改写模型名称（如 minimaxai/minimax-m2.7）
  • 改写接口地址和密钥
  • 可选：调整 system_prompt 和最大重试次数
Step 3 - 启动并使用

点击左侧【启动代理】按钮，然后在客户端中将 API 地址设为：

http://127.0.0.1:8999/v1

客户端的所有请求将通过代理转发，拒答时自动改写重试。

7常见问题
需要安装 Python 吗？
不需要。EXE 绿色版通过 PyInstaller 打包，双击即可运行，所有依赖已内嵌。如需从源码运行则需 Python 3.10+ 环境。
改写模型和目标模型可以用同一个吗？
技术上可以，但不推荐。目标模型已经对某些问题拒答了，用同一个模型改写效果可能不佳。建议使用一个更开放、限制更少的模型作为改写模型。
配置修改后需要重启代理吗？
是的。修改配置后先点击【保存配置】，然后停止再重新启动代理，新配置才会生效。
日志在哪里查看？
两种方式：1) GUI 的"运行日志"选项卡实时查看；2) 打开程序同目录下的 proxy.log 文件。日志每次启动代理时会清空重建。
支持哪些客户端？
所有兼容 OpenAI API 格式的客户端都可以使用，包括 ChatGPT-Next-Web、LobeChat、Open WebUI、Cursor、继续 等。只需将 API 地址指向 http://127.0.0.1:8999/v1 即可。
为什么最小化后找不到窗口了？
窗口最小化时会自动缩到系统托盘（右下角任务栏通知区域）。双击托盘图标可恢复窗口，右键托盘图标可看到快捷菜单。
max_retries 设置多少合适？
默认 5 次已足够覆盖大多数场景。设置过高会增加延迟和 API 开销。如果 5 次改写仍然无法通过拒答过滤，建议检查拦截词列表是否过于宽泛，或优化 system_prompt。
